## 一、安装与启动

### 1.1 普通用户安装（推荐）

从项目 [Releases](https://github.com/jzh-ff/OrangeRadio/releases) 下载对应平台的安装包：

| 系统 | 安装包 | 说明 |
|------|--------|------|
| **Windows x64** | `OrangeRadio_*_x64-setup.exe` | NSIS 安装器，约 6 MB，**推荐**（安装快、可选安装目录） |
| **Windows x64** | `OrangeRadio_*_x64_en-US.msi` | MSI 安装器，约 8.6 MB，企业/批量部署友好 |
| macOS | （v0.4 计划中） | — |
| Linux | （v0.4 计划中） | — |

**Windows 安装步骤**：

1. 双击 `OrangeRadio_*_x64-setup.exe`
2. 选择安装目录（默认 `C:\Program Files\OrangeRadio`）
3. 选择「是否创建桌面快捷方式」「是否加入开始菜单」
4. 点击「安装」，等待约 10 秒
5. 启动器自动从开始菜单启动

> ⚠ **Windows SmartScreen 警告**：当前安装包未做代码签名（计划 v0.5 接入 EV 证书）。首次安装若弹出「Windows 已保护你的电脑」，点击「更多信息」→「仍要运行」即可，不会真的有问题。

### 1.2 开发者本地启动

```powershell
# 一键构建 + 启动（自动配 MSVC link.exe）
.\run.ps1
```

需要本地装好 Rust（≥ 1.96 MSVC toolchain）和 Node.js 18+。首次启动会自动：
- `npm install`（如未装）
- `npm run build`（前端）
- `cargo build -p orangeradio-desktop`（Rust）
- 启动 `target\debug\orangeradio-desktop.exe`

### 1.3 第一次启动

首次启动会自动创建数据目录（Windows 下 `%APPDATA%\com.orangeradio.app\`），无需手动干预。

## 三、设置（齿轮）

点击左下角齿轮按钮打开设置。所有配置**仅存本地**，不进 git、不上传。

### 3.1 🔑 第三方账号登录状态

总览 8 个音源的登录情况：
- **网易云音乐** / **QQ 音乐**：点「登录 / 重新登录」跳到对应音源页，可用 Cookie 或扫码
- **Spotify**：点「配置」输入 Client ID + Secret（来自 [developer.spotify.com](https://developer.spotify.com/dashboard)），1 小时自动续 token
- **酷狗 / 汽水 / 歌曲宝 / 播客**：免登录，显示状态点
- Spotify 已配置后还多一个「清空」按钮

### 3.2 🧠 我的听歌画像

只读展示：
- **总听歌时长**（小时/分钟）
- **常听艺人 Top 8**（带权重百分比）
- **常听流派 Top 8**

画像驱动「懂你模式 🧠」（侧边栏智能分组入口）。多听几首歌画像会更准。

### 3.3 🤖 AI 配置（MiniMax）

| 配置项 | localStorage key | 默认值 | 说明 |
|--------|------------------|--------|------|
| **API Key** | `orangeradio_minimax_key` | （空）| 必填。在 [MiniMax 开放平台](https://platform.minimaxi.com) 注册获取。点击输入框后保存有 1.5s「已保存」提示 |
| **API Base** | `orangeradio_minimax_base` | `https://api.minimaxi.com/anthropic` | 用于写词 + 歌词译注的 Anthropic 兼容端点 |
| **Model** | `orangeradio_minimax_model` | `MiniMax-M1` | LLM 模型名 |

> 💡 这三项同时驱动 **AI 写词**（创作台）和 **歌词译注**（全屏播放页）。

### 3.4 🎵 AI 音乐生成（Studio 创作台）

与上方共用同一个 API Key：

| 配置项 | localStorage key | 默认值 | 说明 |
|--------|------------------|--------|------|
| **Music Base** | `orangeradio_minimax_music_base` | `https://api.minimaxi.com` | MiniMax 音乐生成端点 |
| **Music Model** | `orangeradio_minimax_music_model` | `music-2.6-free` | 限免版（有 RPM 限制）；额度耗尽可切 `music-2.6` 正式版 |

### 3.5 📁 创作输出目录

控制 **OrangeStudio 生成的音频 / 歌词 / 分轨 / 工程文件** 落盘位置：

- **默认（留空）**：`{app_data_dir}/studio/`（Windows 下 `%APPDATA%\com.orangeradio.app\studio\`）
- **自定义**：点「选择…」挑一个文件夹

操作按钮：
- **选择…** — 原生文件夹选择器
- **清除（用默认）** — 清空配置，回退默认
- **打开** — 用系统资源管理器定位当前配置目录

### 3.6 ℹ️ 关于

展示版本号、数据目录、鉴权存储位置、日志路径（只读）。

## 四、播放：本地库与第三方音源

### 4.1 本地音乐库

**首次扫描**：
1. 侧边栏「我的音乐库」
2. 工具栏点「扫描本地音乐」
3. 弹出文件夹选择对话框，多选你要扫的根目录
4. 后台扫描 → 提取元数据（FLAC/WAV/ALAC/MP3/AAC/OGG）→ 入库

支持的元数据：标题、艺术家、专辑、封面、时长、采样率、比特率。封面从音频内嵌提取并缓存到 `covers/`。

### 4.2 第三方音源

| 音源 | 状态 | 登录方式 | 备注 |
|------|------|----------|------|
| **本地库** | ✅ | 无 | — |
| **网络电台** | ✅ | 无 | 调 `radio_popular` / `radio_search` |
| **网易云音乐** | ✅ | Cookie / 扫码 | 支持排行榜 + 每日推荐；登录态后台自动续期 |
| **QQ 音乐** | ✅ | Cookie / 扫码 | 四步换证自动完成 |
| **酷狗音乐** | ✅ | 无 | 免登录搜索 + 播放 |
| **汽水音乐** | 🟡 Alpha | 无 | UI 已上线，接口尚未完全接入 |
| **Spotify** | 🟡 30s | Client ID + Secret | 免费版仅 30s 试听，完整播放需 Premium + Playback SDK |
| **歌曲宝** | ✅ | 无 | 第三方聚合音源，**仅供学习研究**（实际音频来自酷我等平台） |
| **播客** | ✅ | 无 | RSS 订阅 |

### 4.3 网易云 / QQ 音乐登录

#### 网易云

**方式 A · 扫码登录**：
1. 进入「网易云音乐」子视图
2. 点「扫码登录」→ 展示二维码
3. 用网易云 APP 扫码确认
4. 后台轮询（约 2s 一次）等待确认
5. 登录态自动加密存本地，下次启动恢复

**方式 B · Cookie 登录**（扫码失败或风控 8821 时）：
1. 浏览器登录 music.163.com
2. F12 → Application → Cookies → 找到 `MUSIC_U`
3. 复制值，粘贴到 OrangeRadio 的 Cookie 输入框

#### QQ 音乐

类似网易云，扫码 + Cookie 双方式。Cookie 需要 `uin` + `qqmusic_key` 两个。

### 4.4 网易云排行榜 + 每日推荐

「网易云音乐」子视图顶部有：
- **每日推荐**（调用 `netease_daily`）
- **官方排行榜**（调用 `netease_toplists`，封面网格展示）
- 点进单榜 → `netease_toplist_detail` 查看曲目

## 六、歌单与收藏

### 6.1 本地歌单

**新建**：侧边栏「我的歌单」标题栏点 **+** → 弹窗输入名字 → 回车。

**添加曲目**：任意歌曲行点 **＋** → 弹窗「添加到歌单」→ 选目标歌单（弹窗分本地/网易云/QQ 三个 tab）。

**查看**：侧边栏点歌单名 → 进入 `UserPlaylistView`：
- 头部有「▶ 播放全部」按钮
- 曲目列表支持虚拟滚动（长歌单流畅）
- 弹窗内每行有「移除」按钮（不会从源数据删，只从该歌单移除）

**支持跨源**：网易云 / QQ / Spotify 的歌曲可加到本地歌单，播放时按源自动取流。

> 🟡 **v0.5 计划**：歌单重命名 / 删除功能（后端命令已实现，前端 UI 未暴露）。

### 6.2 收藏

| 来源 | 操作 | 行为 |
|------|------|------|
| 本地 | 点 ❤️ | 写本地 `library.liked` 表 |
| 网易云 | 点 ❤️ | 调用 `netease_like_track` 远端收藏 |
| QQ / Spotify 等 | — | 暂未支持 |

底部播放栏 ❤️ 与卡片行 ❤️ 同步状态。

### 6.3 「添加到歌单」弹窗

点 PlayerBar 的「➕」（ListPlus 图标）打开：
- **本地** tab：列本地所有歌单，支持「创建并添加」
- **网易云** tab：列网易云账户歌单（未登录显示提示卡）
- **QQ** tab：🟡 v0.5+ 占位

## 七、全屏沉浸播放页

### 7.1 入口

- 底部播放栏左侧**封面**点击
- 侧边栏底部 **NOW PLAYING** 条点击
- 全局/局内热键 `F`

### 7.2 四种布局

顶部导航条切换：

| id | 名称 | 说明 |
|----|------|------|
| 🎬 **cinema** | 电影粒子 | 全屏 Three.js 节拍粒子 + 中央大字歌词（舞台扫光） |
| 🖼 **immersive** | 沉浸双栏 | 左封面 + 右歌词滚动 |
| 📜 **lyric-stream** | 歌词流 | 全屏居中超大歌词（阅读优先） |
| ▦ **triple** | 三栏详情 | 歌词 + 封面 + 网易云/QQ 热门评论 |

### 7.3 退出

- 按 `Esc`
- 或右上角 ✕

### 7.4 cinema 模式视觉控制台

右上角齿轮打开「Visual Console」：
- 节拍灵敏度 / 粒子数量 / Bloom 强度
- 颜色主题：橙焰 / 电紫 / 深海 / 极光
- 镜头晃动开关

参数持久化到 localStorage。

### 7.5 热门评论

cinema 模式右下角 💬 按钮打开评论抽屉；`triple` 模式右侧常驻。网易云 + QQ 音乐支持（头像/昵称/内容/点赞数）。

### 7.6 译注按钮（🔮）

顶部导航右侧的「译注」按钮：
- 灰色「译注」= 未译注，点击调 AI 翻译当前歌词
- 高亮「译注 ✓」= 已译注，点击取消显示
- 「…」 = 译注中

详见 [第九节](#九ai-歌词译注)。

## 八、桌面歌词悬浮窗

独立的、**始终置顶、可穿透鼠标**的悬浮歌词窗口。

### 8.1 打开方式

- 底部播放栏右侧「**词**」字按钮（最近的视觉打磨把图标换成了文字更直观）
- 或局内热键 `Alt+L` / 全局热键 `Ctrl+Alt+L`

### 8.2 窗口特性

| 特性 | 值 |
|------|-----|
| 尺寸 | 1100 × 260 px |
| 装饰 | 无边框、背景透明 |
| 置顶 | 始终 |
| 任务栏 | 不显示 |
| 可移动 | 是（拖动） |
| 穿透鼠标 | 是（点击穿透到下方窗口） |

### 8.3 显示内容

- **当前行**大字扫光高亮
- 上 / 下各 2 行上下文（共 5 行）
- 等宽或艺术字字体（主题决定）

### 8.4 控制条（hover 显）

- 上一首 / 播放暂停 / 下一首
- ⚙ 设置（字号系数 0.7~1.5 / 不透明度 0.4~1.0 / 翻译开关 / 主题：默认/暖橙/冷蓝/纯白）
- 🔒 **锁定**：开启后鼠标穿透到底层窗口，方便边看歌词边做其他事；锁定状态下「词」字按钮变橙色提示

## 九、AI 歌词译注

给当前播放的歌词做 AI 翻译 + 背景解读 + 行级注解。

### 9.1 触发

全屏播放页右上「🔮 译注」按钮：
- 必须当前歌曲有歌词（本地内嵌 LRC / 网易云 LRC / QQ LRC）
- 必须设置页配置了 MiniMax API Key

### 9.2 输出

返回 JSON 结构：

```json
{
  "background": "这首歌写于 1987 年的东柏林…",
  "lines": [
    { "original": "Yesterday, all my troubles…",
      "translation": "昨日，我所有的烦恼…",
      "annotation": "Beatles 1965 年的经典…"
    }
  ]
}
```

### 9.3 展示位置

- **顶部「AI 背景」卡片**：cinema 模式铺在舞台上方，非 cinema 模式在顶部固定
- **行级注解**：每行歌词下方可展开/收起（点 ↕ 按钮）

### 9.4 计费

调一次 MiniMax LLM，约 2-5 秒、消耗 1k-3k tokens（取决于歌词长度）。

## 十、OrangeStudio AI 音乐创作

> 这是 OrangeRadio 的核心差异化能力。由 **MiniMax music-2.6** 驱动。
>
> 完整的技术文档：[`06-创作工作室指南.md`](06-创作工作室指南.md)

### 10.1 进入方式

侧边栏顶部主模块切换到「🎚 创作工作室」。

### 10.2 前置：配置 API Key

**首次使用必须**：
1. 打开设置（齿轮）
2. 「🤖 AI 配置（MiniMax）」填 API Key
3. 「🎵 AI 音乐生成（Studio 创作台）」可改 Music Model
4. 「📁 创作输出目录」可改落盘位置

未配置时，输入框下方会有黄色提示「⚠ 未配置 MiniMax API Key，请先点击左下角齿轮设置」。

### 10.3 四步工作流

#### 步骤 ① 描述你想要的歌

输入风格描述（自然语言），例如：

> 一首 80 年代复古合成器流行，关于夏夜海边的回忆，欢快中带点怀旧，BPM 110

也可点下方 3 个示例 prompt 一键填充。

点「**开始创作**」直接调音乐生成（**会自动写词**，见下）。

#### 步骤 ② AI 写词（可选）

如果想先看 / 编辑歌词，再生成音乐：

1. 点「**AI 写词**」（或「重新写词」）
2. 约 3-8 秒后返回一版结构化歌词（主歌 / 副歌 / 桥段）
3. 弹出可编辑 textarea，可手动改任何行
4. 主题 + 押韵方案显示在副文案

> 💡 **自动写词**：步骤 ① 如果你**直接点开始创作**（没手动写词），后端会自动调一次 LLM 写词并同时用于音乐演唱和回传展示；这样你听得到、也看得到。

#### 步骤 ③ 生成结果

等待约 30-90 秒（同步接口，MiniMax 生成中）。

完成后展示：
- 🎵 `<audio>` 原生播放器（autoplay）
- 📂 **打开目录** — 系统资源管理器打开输出目录
- 📜 **查看歌词 / 隐藏歌词** — 展开折叠的歌词文本
- 底部显示本地缓存路径

输出目录每个产物：
- `{timestamp}-{task_id}.mp3` — 音频
- `{timestamp}-{task_id}.txt` — 歌词（如有自动写词）
- `{timestamp}-{task_id}.orp` — 工程文件（如保存过）

#### 步骤 ④ 人声 / 伴奏分轨

点「**生成分轨**」：

> ⚠ 会调用 MiniMax **两次**（带唱 + 纯伴奏），消耗**双倍额度**。两次生成基于同一 prompt 但**旋律/编曲会有随机差异**（适合试听，非精确分离）。

完成后两个播放器：
- 🎤 人声版（带唱）
- 🎸 纯伴奏版（instrumental）

### 10.5 关闭

在步骤 ① 把 prompt 清空 + 步骤 ③ 把 audioPath 清掉（点几次会触发 reset）。

## 十一、一起听（同步房间）

和朋友用同一个房间号，**实时同步播放进度**。

> 🟡 **状态**：WebSocket 同步逻辑已上线；侧边栏「一起听」入口 / 独立社交中心页是 v0.7 占位。
> 当前入口在底部播放栏的「一起听」按钮。

### 11.1 使用方式

1. **底部播放栏点「一起听」图标**（ListenTogether）
2. 弹窗输入房间号（任意字符串，默认 `"default"`，和朋友用同一个字符串即可）
3. 进入房间后按钮变橙色
4. 你的播放 / 暂停 / 切歌 / 拖进度会实时同步给房间其他人
5. 离开再点一次，输入空或新房间号

### 11.2 后端

需要先启动 `server/` crate：

```bash
cd server && cargo run
# 默认 ws://localhost:3847
```

前端默认连这个地址（`frontend/src/lib/listenTogether.ts:20`）。

## 十三、数据目录与备份

### 13.1 数据目录位置

| 系统 | 路径 |
|------|------|
| Windows | `%APPDATA%\com.orangeradio.app\` |
| macOS | `~/Library/Application Support/com.orangeradio.app/` |
| Linux | `~/.local/share/com.orangeradio.app/` |

### 13.2 目录结构

```
{app_data_dir}/
├── studio/                # 默认创作输出目录（可改）
│   ├── {ts}-{task_id}.mp3
│   ├── {ts}-{task_id}.txt
│   └── {ts}-{task_id}.orp
├── covers/                # 本地音乐封面缓存（FNV 哈希命名）
├── cache/                 # QQ 音乐流缓存
├── logs/                  # 日志（按天滚动）
├── auth/                  # 登录态（AES-256-GCM 加密 *.bin）
└── (其他 Tauri 自动文件)
```

数据库（库 + 歌单 + 收藏 + 听歌画像）默认在系统约定位置（`{app_data_dir}` 下 `library.sqlite` 或类似）。

### 13.3 备份建议

- **本地库**：`covers/` 整个目录
- **歌单 + 收藏**：数据库文件（找到 `library.sqlite` 备份）
- **创作产物**：`studio/` 整个目录（或你自定义的输出目录）

### 13.4 完全卸载

Windows：
- 设置 → 应用 → OrangeRadio → 卸载
- 数据目录 `%APPDATA%\com.orangeradio.app\` 不会被卸载程序自动删，**手动备份后再删**

## 十四、常见问题

### Q1: 网易云歌曲播放失败？

可能原因：
- 歌曲需 VIP 权限
- Cookie 过期（侧边栏重新登录）
- 风控 code 8821（自动切到 Cookie 登录）

### Q2: QQ 音乐播放没声音 / 几小时后失效？

QQ 的 vkey 有时效（约数小时）。长时间不操作后失效，**重新搜索或切歌**即可获取新 vkey。

### Q3: Spotify 只有 30 秒？

Spotify 免费版仅 30s 试听。完整播放需要 Premium 账号 + Playback SDK（v0.5 计划）。

### Q4: 本地音乐没封面？

OrangeRadio 会从音频文件内嵌提取封面（FLAC/M4A/MP3 都支持）。如果文件本身没有内嵌封面，会显示橙色占位。

### Q5: AI 写词 / 音乐生成报错？

最常见：
- **未配置 API Key**：设置 → 🤖 AI 配置 填 Key
- **API Key 无效 / 余额不足**：去 [MiniMax 控制台](https://platform.minimaxi.com/user-center/basic-information/interface-key) 看
- **网络问题**：科学上网 / 检查 base URL 是否正确（默认 `https://api.minimaxi.com`）

### Q6: 桌面歌词窗口打不开 / 显示异常？

- 检查底部播放栏的「词」字按钮是否处于锁定（橙色）
- 锁定状态下鼠标穿透，会让按钮难以点击 → 再点一次解锁
- 窗口位置可能飘出屏幕外 → 关闭重开会复位到默认位置

### Q7: 安装包提示「Windows 已保护你的电脑」？

代码签名未做（v0.5 计划 EV 证书）。点击「更多信息」→「仍要运行」即可，不会真的有问题。

### Q8: 一起听按钮在哪？

侧边栏「一起听」入口**当前是 v0.7 占位**。使用入口在**底部播放栏的「一起听」图标**。同时需要后端 `server/` crate 在 `localhost:3847` 运行。

### Q9: 创作输出目录怎么改？

设置 → 📁 创作输出目录 → 「选择…」挑一个文件夹。重启应用生效（实际上立即生效，但建议重启保险）。

### Q10: 怎么开一起听 / DLNA 投屏？

- 一起听：v0.7 上线独立社交中心页（当前已在 PlayerBar 提供）
- DLNA / AirPlay / Chromecast：v0.8 计划

### Q11: 汽水音乐能搜到但点不动？

汽水音乐（Alpha）接口尚未完全接入，搜索可用但播放可能失败。等 v0.5 完善。

### Q12: 全局热键冲突？

可到设置 → 热键自定义（localStorage `orangeradio-global-hotkeys-v1`）。
