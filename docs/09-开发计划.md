## v0.3 音源生态（下一个迭代）

**目标**：让用户能听网络电台、播客、第三方平台音乐。

| # | 任务 | 优先级 | 依赖 | 估时 |
|---|------|--------|------|------|
| 1 | 本地库 SQLite 持久化（替代内存索引） | P0 | — | 1d |
| 2 | 真实频谱：Rust 端 FFT 通过 IPC 推前端 | P0 | — | 2d |
| 3 | 网络电台接入（RadioBrowser API） | P0 | AudioSource trait | 2d |
| 4 | 播客 RSS 订阅 + episode 解析 | P1 | AudioSource trait | 2d |
| 5 | 音源插件 trait 完善与动态加载 | P0 | — | 1d |
| 6 | 网易云音乐接入（第三方） | P1 | 插件 trait | 3d |
| 7 | QQ 音乐接入（第三方） | P2 | 插件 trait | 3d |
| 8 | Spotify 官方 API | P2 | OAuth2 | 2d |
| 9 | 多源聚合搜索 UI | P1 | 音源就绪 | 1d |

**关键依赖链**：`AudioSource trait 完善(5) → 各音源实现(3/4/6/7/8) → 聚合搜索(9)`

**合规边界**（重要）：
- 第三方平台（网易云/QQ）只用用户已登录账号，不破解付费墙
- 不持久化音源字节流，只缓存元数据
- 优先走官方 API（Spotify/Apple 有官方）
- 本地库 + 网络电台 + 播客是稳定合规基础，第三方作"补充"

## 跨版本技术债（持续处理）

| 债务 | 当前状态 | 解决版本 |
|------|----------|----------|
| 频谱是模拟数据 | CORS 限制无法用 MediaElementSource | v0.3 Rust FFT |
| 本地库每次重扫 | 内存索引无持久化 | v0.3 SQLite |
| GNU 工具链 | WinLibs GCC，非官方推荐 MSVC | v1.0 评估 |
| 前端 chunk 过大 | Three.js 单包 977KB | 优化期 code-split |

## 排期原则
1. 每个 P0 必须做完才能进下一版本
2. P1 尽量在当前版本做完，可延后
3. P2 可延后到后续版本或社区贡献
4. 遇到合规风险（第三方音源）优先稳态方案（本地+电台+播客）
