# 03 - OrangeRadio 开发文档

> 面向开发者 · v0.2 · 最后更新 2026-07-01

## 一、环境搭建（Windows，实测可用）

> ⚠️ 这是最容易踩坑的部分。请严格按顺序安装。

### 1. Rust 工具链（GNU）

```powershell
# 安装 rustup
winget install Rustlang.Rustup

# 安装 GNU 工具链（不要用 msvc，机器没有 Build Tools）
rustup toolchain install stable-gnu
rustup default stable-gnu
```

验证：`rustc --version` 应输出 `rustc 1.9x.0 (xxx 2026-xx-xx)` 且包含 `gnu`。

### 2. WinLibs MinGW-w64（关键！）

Rust GNU 工具链需要 GCC-based 的 MinGW 提供 `libgcc`：

```powershell
winget install BrechtSanders.WinLibs.POSIX.MSVCRT -e --accept-package-agreements --accept-source-agreements
```

⚠️ **必须把 WinLibs 的 bin 目录加到 PATH 最前面**，否则 Rust 会找到系统里其他的 gcc（如 LLVM-MinGW），导致链接报错 `unable to find library -lgcc_eh`。

默认安装路径：
```
%LOCALAPPDATA%\Microsoft\WinGet\Packages\BrechtSanders.WinLibs.POSIX.MSVCRT_Microsoft.Winget.Source_8wekyb3d8bbwe\mingw64\bin
```

**本项目 `run.ps1` 已自动处理 PATH**，手动编译时需自行 export。

### 3. Node.js

```powershell
# 已有 Node 18+ 可跳过
winget install OpenJS.NodeJS
```

验证：`node --version` ≥ 18。

### 4. 前端依赖

```powershell
cd D:\ZCodeWP\OrangeRadio\frontend
npm install
```

## 二、一键启动

```powershell
cd D:\ZCodeWP\OrangeRadio
.\run.ps1
```

脚本会自动：① 设置 WinLibs PATH → ② 构建前端 → ③ 编译 Rust → ④ 启动应用。

## 三、手动分步操作

### 构建前端
```powershell
cd frontend
npm run build      # 产物输出到 ../apps/desktop/dist/
```

### 编译 Rust（需先 export WinLibs PATH）
```powershell
$env:PATH = "<WinLibs路径>\mingw64\bin;$env:PATH"
cargo build -p orangeradio-desktop
```

### 启动
```powershell
.\target\debug\orangeradio-desktop.exe
```

### 启动社交后端（独立）
```powershell
cargo run -p orangeradio-server    # → http://localhost:3847
```

## 四、常见报错排查

| 报错 | 原因 | 解决 |
|------|------|------|
| `unable to find library -lgcc_eh / -lgcc` | WinLibs 不在 PATH，或找到了 LLVM-MinGW | 把 WinLibs bin 加 PATH 最前；用 `run.ps1` |
| 应用白屏 | vite `base: "./"` 配置 | 已修复，不要加 `base` |
| 播放没声音 | `createMediaElementSource` CORS 静音 | 已修复，只用 `<audio>` 不接 MediaElementSource |
| `frontend\npm : 无法加载模块` | PowerShell 把 `frontend\npm` 当命令 | 先 `cd frontend` 再 `npm`，或用 `run.ps1` |
| `.ps1` 脚本报「字符串缺少终止符」 | PS5 按 GBK 解码 UTF-8 脚本 | 脚本已改纯英文 |

## 六、核心数据流

**播放（用户双击歌曲）：**
1. `LibraryView` → `engineRef.playTrack(track, index)`
2. `useAudioEngine.playPath()` → `convertFileSrc()` → `<audio>.play()`
3. audio 事件 → `playerStore`（position/duration/isPlaying）
4. `VisualBackground` 读 `playerStore.spectrum` 渲染粒子

**扫描（用户点扫描）：**
1. `libraryStore.scanLocal()` → 弹窗选目录
2. `invoke("library_scan")` → Rust `LibraryScanner.scan()`
3. walkdir 遍历 + lofty 读元数据 → `LibraryDb` 内存索引
4. 返回前端 → `libraryStore.setTracks()` + `playerStore.setQueue()`

## 七、新增 IPC 命令

在 `crates/orange-tauri/src/commands.rs` 添加：
```rust
#[tauri::command]
pub async fn my_command(state: tauri::State<'_, AppState>, arg: String) -> Result<String, String> {
    Ok("done".into())
}
```
并在 `register_all()` 的 `generate_handler![]` 里注册。

前端调用：
```typescript
const result = await invoke<string>("my_command", { arg: "value" });
```

## 八、新增前端功能

- 状态放 `stores/`（Zustand）
- UI 组件放 `features/<模块>/`
- 全局引擎调用通过 `engineRef`（在 App.tsx 导出）
