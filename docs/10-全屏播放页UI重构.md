## 一、重构背景与目标

用户反馈四个问题，本次一次性解决：

| # | 问题 | 根因 | 修复策略 |
|---|---|---|---|
| 1 | 视觉控制台在低分辨率下展示不完全且无法滚动 | `.vc-tab-content { max-height: 60vh }` 在抽屉内嵌时未被覆盖 + 双层嵌套滚动 | 解除 60vh 封顶，单一滚动源 |
| 2 | 右侧抽屉纯 hover 打开，调参时鼠标离开会误关 | 无钉住态，纯 hover + 250ms 延迟关闭 | 点击切换 + 钉住 + 悬浮按钮 |
| 3 | 专辑旋转时背部有固定方形框 | `.fp-cover-big--live` 的 `fp-cover-pulse` 给不旋转的方形容器加描边/投影 | 圆形黑胶：封面图 `border-radius:50%` + 容器透明 |
| 4 | 五种布局实际效果与命名差距大；评论区只能看一点 | 命名不准确 + 评论区 `max-height:calc(100vh-240px)` 硬编码 + 嵌套 overflow | 重设计 + 修正命名 + 修复高度 |

## 三、架构改动详解

### 3.1 右侧抽屉交互重构

**文件**：`features/player/FullPlayerRightDrawer.tsx` + `styles/right-drawer.css`

#### 交互状态机

```
状态：open（是否展开）+ pinned（是否钉住）

初始：open=false, pinned=false

[悬浮按钮点击] → togglePinned()
  - 清掉残留 hover 定时器（防竞态）
  - pinned = !pinned; open = pinned

[未钉住时 hover] → onHoverEnter/onHoverLeave
  - if (pinned) return  ← 钉住态 hover 不干扰
  - 250ms 延迟收回
```

**关键修复**：`togglePinned` 必须清 hover 定时器，否则"hover 预览后立即点击钉住"会被 pending 的收回定时器意外关闭。

#### 悬浮按钮（FAB）定位

```
关闭态：position:fixed; right:0; top:50% （屏幕最右侧垂直居中）
打开态：right:360px （移到 panel 左边缘外，作为把手露出，不遮挡内容）
```

箭头随开合翻转（`◀` → `▶`），文字竖排"工具"。

### 3.2 控制台滚动修复

**文件**：`styles/right-drawer.css`

```css
/* 修复前：max-height:60vh 封顶 + 双层 overflow 嵌套 */
.fp-rd__vc-host .vc-tab-content { flex:1; min-height:0; overflow-y:auto; }

/* 修复后：解除封顶，单一滚动源（外层 panel 滚） */
.fp-rd__vc-host .vc-tab-content {
  flex: none;
  max-height: none !important;
  min-height: 0;
  overflow: visible;  /* 不自滚，由 .fp-rd__panel 统一滚 */
}
```

### 3.3 圆形黑胶封面

**文件**：`styles/full-player.css`

```
DOM 层级：.fp-cover-frame（透明容器）
         └ .fp-cover-big（圆形容器，border-radius:50%，透明，持有发光晕）
           └ .fp-cover-img（旋转层，border-radius:50%，24s linear infinite）

发光晕：挂在 .fp-cover-big--live 上，box-shadow 圆形对称，不随旋转变形
中心孔：.fp-cover-big::after，覆盖在旋转封面图中心（z-index:3）
```

**为何不用 img::after**：`<img>` 是替换元素，其 `::before/::after` 伪元素不渲染（CSS 规范）。中心孔必须挂在非替换元素 `.fp-cover-big` 上。

### 3.4 五种布局重新设计

| 布局 | 旧名 → 新名 | 设计定位 |
|---|---|---|
| rhythmic-album | 律动专辑 → 律动专辑 | 粒子方阵 + 中心歌词舞台 |
| rhythmic-particles | 粒子律动 → 粒子律动 | 粒子背景 + 电影运镜 |
| immersive | 沉浸双栏 → **沉浸黑胶** | 左 280px 圆形黑胶 + 右歌词流，`NOW PLAYING` 刊眼 |
| lyric-stream | 歌词流 → **歌词长卷** | 顶部紧凑黑胶横条 + 下方歌词吃满高度 |
| triple | 三栏详情 → **杂志三栏** | 黑胶/歌词/评论三栏等高，栏间香槟金发丝线，`VOICES · 热评` 刊眼 |

### 3.5 评论区高度修复

**文件**：`styles/full-player.css`

```css
/* 修复前：硬编码 max-height + 嵌套 overflow */
.fp-comments-section { max-height: calc(100vh - 240px); overflow-y: auto; }
.cl-list { overflow-y: auto; }  /* 嵌套滚动 */

/* 修复后：flex 撑满 + 单一滚动源 */
.fp-comments-section { flex:1; max-height:100%; overflow-y:auto; }
.cl-list { padding: 8px 18px; }  /* 不自滚 */
```

**响应式（≤900px 垂直堆叠）**：评论区改为 `max-height:40vh`（去掉对父级 100% 依赖，防塌陷）。

## 六、手动验收指引

启动 `cargo tauri dev` 后依次验证：

1. **抽屉交互**：点击右侧"工具"悬浮按钮 → 抽屉打开并钉住 → 鼠标移开不会收回 → 在视觉控制台调滑块 → 再次点击按钮收起。
2. **封面黑胶**：切到沉浸黑胶布局 → 封面为圆形旋转 → 无方形描边/投影 → 中心有黑胶孔 → 暂停时停转。
3. **评论区**：切到杂志三栏 → 右侧评论可见多条 → 可滚动 → 窗口缩到 900px 以下 → 评论区不塌陷。
4. **控制台滚动**：在低分辨率（或缩放窗口）打开抽屉 → 视觉控制台所有 tab 内容可滚动看全。
5. **五种布局**：逐一切换 → 命名与效果对应（沉浸黑胶/歌词长卷/杂志三栏）。

## 七、已知遗留与后续优化

| 项 | 说明 | 优先级 |
|---|---|---|
| `.fp-rd__vc-host .vc-panel` 的 `!important` 批量覆盖 | 维护脆弱，建议给 VisualConsole 加 `variant="inline"` prop 用不同类名隔离 | 低 |
| `--fp-footer-h` 定义在 `.fp-overlay` 上被 `.fp-rd` 依赖 | 若抽屉移出 `.fp-overlay` 需同步迁移该变量 | 低 |
| 黑胶中心孔在无封面图分支会盖住 OrangeRadioLogo | 可用 `:has(.fp-cover-img)` 或修饰类控制 | 低 |
