# 11 - 播放队列优化技术文档

> 本文档记录 OrangeRadio v0.3 阶段对播放队列系统的全面优化，包括双队列隔离、虚拟化、持久化、增量 API、分页和 3D 歌单架增强。

## 1. 背景与问题

优化前的播放队列存在以下问题：

| 问题 | 影响 |
|------|------|
| **无虚拟化** | 所有列表 `tracks.map(...)` 全量渲染，本地库上万首时卡顿 |
| **队列无持久化** | 重启/刷新后 `tracks` 清空，需手动重新进入视图 |
| **`setQueue` 只能整体替换** | 无法做「下一首播放」「添加到队列」等基础操作 |
| **电台与单曲混在同一队列** | 电台流（无时长）和单曲语义冲突，切 view 互相覆盖 |
| **电台 source_kind bug** | `web_radio.rs::to_track` 未设 source_kind，默认 `local`，前端无法区分 |
| **QQ 歌单卡片无 onClick** | 卡片纯展示，点击无反应（后端 detail 能力齐全） |
| **3D 歌单架是 MVP** | 黑胶占位、无交互相机、无 hover、无动画 |

## 2. 双队列隔离模型

### 2.1 数据结构

`playerStore` 拆分为两个独立队列 + 一个活跃标记：

```ts
interface PlayerState {
  tracks: Track[];        // 单曲队列（网易云/QQ/本地/gequbao/播客/Spotify）
  currentIndex: number;
  radioTracks: Track[];   // 电台队列（独立）
  radioIndex: number;
  activeQueue: "tracks" | "radio";  // 驱动 next/prev 路由
}
```

### 2.2 路由机制

`useAudioEngine.next/prev` 根据 `activeQueue` 选择操作的队列：

```ts
const isRadio = st.activeQueue === "radio";
const list = isRadio ? st.radioTracks : st.tracks;
const idx = isRadio ? st.radioIndex : st.currentIndex;
```

- **RadioView** 调 `setRadioQueue()`（写 radioTracks + activeQueue:"radio"）
- **其他视图** 调 `setQueue()`（写 tracks + activeQueue:"tracks"）
- **懂你模式**（understand_you）仅单曲队列生效，电台不触发推荐

### 2.3 前置修复

`crates/orange-sources/src/web_radio.rs::to_track` 补：

```rust
track.source_kind = SourceKind::WebRadio;
```

否则前端任何 `source_kind === "web_radio"` 判断失效（Track::new 默认 Local）。

## 3. 队列持久化

用 zustand `persist` 中间件，仅持久化低频字段：

```ts
persist(
  (set, get) => ({ /* store 实现 */ }),
  {
    name: "orangeradio_player",
    storage: createJSONStorage(() => localStorage),
    partialize: (s) => ({
      tracks: s.tracks, currentIndex: s.currentIndex,
      radioTracks: s.radioTracks, radioIndex: s.radioIndex,
      activeQueue: s.activeQueue, mode: s.mode, volume: s.volume,
      currentTrack: s.currentTrack,
    }),
  }
)
```

排除 `spectrum/beat/beatmap` 等高频瞬时态。persist 自动 hydrate，无需手动恢复逻辑。

## 4. 列表虚拟化

### 4.1 技术选型

- **react-window `FixedSizeList`**（非 virtuoso）：9/10 列表行高固定 48px，FixedSizeList 性能最优、API 最简、包体 6KB
- 行高 48px（`.lib-row` CSS 写死，library.css:141）

### 4.2 统一 `<TrackRow>` 组件

抽出 9 个视图共用的 `col-i`（序号/eq-bars/play-hover）逻辑：

```tsx
<TrackRow track={t} index={i} active isPlaying onPlay={fn} cols={5}>
  {/* title/artist/album/dur 列内容由调用方自定义 */}
</TrackRow>
```

- `cols={4}` 用于无专辑列的视图（歌曲宝）
- `cols={5}` 默认（含专辑列）

### 4.3 `<VirtualTrackList>` 包装

自动测量父容器高度 + ResizeObserver：

```tsx
<VirtualTrackList
  tracks={tracks}
  activeId={currentTrack?.id}
  isPlaying={isPlaying}
  onPlay={handlePlay}
  renderRow={(t, i) => (<>...</>)}
  cols={5}
  onItemsRendered={onItemsRendered}  // 无限滚动
/>
```

### 4.4 QueuePanel 虚拟化

`.qp-item` 加固定 `height: 56px`，用 FixedSizeList 分别虚拟化单曲队列和电台队列。

## 5. 增量队列 API

```ts
addToQueue(track)         // 追加到队尾
addManyToQueue(tracks)    // 批量追加
insertNext(track)         // 插入到 currentIndex+1（「下一首播放」）
removeAt(index)           // 删除指定位置（currentIndex 语义正确维护）
clearQueue()              // 清空
```

`insertNext` 实现要点：插入点 ≤ currentIndex 时 currentIndex 后移，保持指向原曲。

## 6. 后端分页

### 6.1 命令层加 page 参数

7 个搜索命令加 `page: Option<u32>`（默认 1，向后兼容）：

| 命令 | page_size |
|------|-----------|
| `search` | 100 |
| `radio_search` | 50 |
| `gequbao_search` | 50 |
| `netease_search` | 30 |
| `qqmusic_search` | 30 |
| `spotify_search` | 20 |
| `library_tracks` | offset+limit（新增分页重载） |

底座（`SearchQuery`/`database.search`/各 source impl）早已支持分页，唯一缺口是命令层硬编码 `page:1`。

### 6.2 前端无限滚动

`useVirtualInfiniteScroll` hook 基于 react-window 的 `onItemsRendered`：

```ts
const onItemsRendered = useVirtualInfiniteScroll({
  hasMore, loading, onLoadMore: loadMore, threshold: 8,
});
```

可见项末尾距列表底部 ≤ threshold 时触发 `loadMore`，追加结果到 tracks + `addManyToQueue`。

**已接入视图（全部 8 个长列表）**：

| 视图 | 数据源 | 分页机制 |
|------|--------|----------|
| LibraryView | `library_tracks(offset, limit)` 每页 200 | 客户端分页（搜索时禁用） |
| SearchView | `search_all(page)` | 聚合各源 page 同步翻页 |
| NeteaseView | `netease_search(page)` | 仅 search 视图分页 |
| QqMusicView | `qqmusic_search(page)` | 仅 search 视图分页 |
| GequbaoView | `gequbao_search(page)` | 仅搜索时分页 |
| SpotifyView | `spotify_search(page)` | 始终分页 |
| RadioView | `radio_search(page)` | 仅搜索时分页 |
| UserPlaylistView | `playlist_tracks(offset, limit)` 每页 100 | 始终分页 |
| PodcastView | `podcast_fetch`（RSS 一次性返回） | 不分页（RSS 协议无分页），已虚拟化 |

### 7.1 后端 all_playlists cover 提取

`crates/orange-library/src/database.rs::all_playlists` 对每个歌单查询第一首有 artwork 的曲目，提取网络封面 URL：

```rust
// extract_cover_url 只返回 Url 类型（Local/Emedded 返回 None）
fn extract_cover_url(a: &ArtworkSource) -> Option<&str> {
    match a {
        ArtworkSource::Url { url } => Some(url.as_str()),
        ArtworkSource::Local { .. } => None,
        ArtworkSource::Embedded { .. } => None,
    }
}
```

`UserPlaylist` 新增 `cover: Option<String>` 字段（`skip_serializing_if = "Option::is_none"`，无封面时不序列化）。
