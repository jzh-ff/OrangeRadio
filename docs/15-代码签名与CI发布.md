# 15 - 代码签名与 CI 发布

> 本文档说明如何为 OrangeRadio 配置 GitHub Actions 三平台自动构建与代码签名。

## 当前状态

> **当前 CI 未启用代码签名**，发布产物为未签名的 `.msi` / `.exe` / `.dmg` / `.deb` / `.AppImage`。
> 本文件保留完整签名配置方法，供后续需要正式分发时启用。

启用签名需要：
1. 按下方说明申请证书 / Azure 资源
2. 在 GitHub Secrets 中填入对应参数
3. 取消 `.github/workflows/release.yml` 中注释掉的签名步骤
4. 在 `apps/desktop/src-tauri/tauri.conf.json` 的 `bundle.windows` 中恢复 `signCommand`

## 一、当前 CI 状态

项目已配置 `.github/workflows/release.yml`，支持以下触发方式：

- 手动触发：GitHub 仓库 → Actions → Release → Run workflow
- 自动触发：推送 `v*` 标签，例如 `git tag v0.3.1 && git push origin v0.3.1`

构建矩阵：

| 平台 | Runner | 默认产物 |
|---|---|---|
| Windows x64 | `windows-latest` | `.msi` / `.exe` (NSIS) |
| macOS Intel | `macos-latest` | `.dmg` / `.app` (x86_64) |
| macOS Apple Silicon | `macos-latest` | `.dmg` / `.app` (aarch64) |
| Linux | `ubuntu-22.04` | `.deb` / `.AppImage` |

## 二、代码签名方案

### macOS：Apple Developer ID + Notary

macOS Gatekeeper 对未签名应用限制最严格，建议优先配置。

#### 所需材料

1. **Apple Developer Program** 会员（$99/年）
2. **Developer ID Application 证书**（用于签名 `.app`）
3. **App Store Connect API Key** 或 Apple ID + App 专用密码（用于公证）

#### 生成 p12 证书

1. 在 Mac 上打开「钥匙串访问」
2. 申请并下载 **Developer ID Application** 证书
3. 导出为 `.p12` 格式，设置密码
4. 转换为 Base64：
   ```bash
   base64 -i Certificates.p12 -o certificate.base64.txt
   ```

#### 设置 Secrets

在 GitHub 仓库 → Settings → Secrets and variables → Actions → Repository secrets 中添加：

| Secret | 说明 |
|---|---|
| `APPLE_CERTIFICATE` | p12 文件 Base64 内容 |
| `APPLE_CERTIFICATE_PASSWORD` | p12 导出密码 |
| `APPLE_ID` | 你的 Apple ID 邮箱 |
| `APPLE_PASSWORD` | Apple ID 专用密码（不是普通登录密码） |
| `APPLE_TEAM_ID` | Apple Team ID，10 位字符 |

> 使用 App Store Connect API Key 替代 Apple ID 的方案见 [Tauri 官方文档](https://tauri.app/distribute/sign/ios/)。

### Windows：Azure Trusted Signing（推荐）

Azure Trusted Signing 是微软现代的云签名服务，比传统 OV/EV 证书更便宜、更易轮换。

#### 前置条件

1. Azure 订阅
2. 在 Azure Portal 中创建：
   - **Trusted Signing Account**
   - **Certificate Profile**
3. 为 GitHub Actions 配置 **Federated credentials**（推荐）或 Service Principal

#### 获取参数

在 Azure Portal → Trusted Signing Account 中获取：

- `Account name`
- `Certificate profile name`
- `Endpoint URL`（例如 `https://eus.codesigning.azure.net`）

#### 设置 Secrets

| Secret | 说明 |
|---|---|
| `AZURE_CLIENT_ID` | Service Principal 的 Application (client) ID |
| `AZURE_TENANT_ID` | Azure AD Tenant ID |
| `AZURE_SUBSCRIPTION_ID` | Azure 订阅 ID |
| `AZURE_TRUSTED_SIGNING_ACCOUNT_NAME` | Trusted Signing Account 名称 |
| `AZURE_TRUSTED_SIGNING_PROFILE_NAME` | Certificate Profile 名称 |
| `AZURE_TRUSTED_SIGNING_ENDPOINT` | Endpoint URL |

> 本项目已配置 `azure/login@v2` 使用 OIDC Federation，无需在 GitHub 中保存 Azure 密码。

### Windows：传统证书（替代方案）

如果你已有 `.pfx` 代码签名证书，可以修改 `apps/desktop/src-tauri/tauri.conf.json`：

```json
"windows": {
  "signCommand": "signtool sign /f ${{ github.workspace }}\\certs\\certificate.pfx /p %CERTIFICATE_PASSWORD% /tr http://timestamp.digicert.com /td sha256 /fd sha256 %1"
}
```

并在 CI 中将 `CERTIFICATE_PASSWORD` 注入为 Secret。

### Linux

Linux 桌面应用通常不需要代码签名即可安装。如需验证来源，可额外对 `.AppImage` 做 GPG 签名，本工作流暂未配置。

## 三、启用签名时的完整 Secrets 清单

如需运行带签名的 Release，需要在 GitHub 仓库 → Settings → Secrets and variables → Actions → Repository secrets 中配置以下 Secrets：

```
APPLE_CERTIFICATE
APPLE_CERTIFICATE_PASSWORD
APPLE_ID
APPLE_PASSWORD
APPLE_TEAM_ID
AZURE_CLIENT_ID
AZURE_TENANT_ID
AZURE_SUBSCRIPTION_ID
AZURE_TRUSTED_SIGNING_ACCOUNT_NAME
AZURE_TRUSTED_SIGNING_PROFILE_NAME
AZURE_TRUSTED_SIGNING_ENDPOINT
```

如果某平台暂时不想签名，可以：

- **跳过 macOS 签名**：不填 `APPLE_*` 相关 Secrets，Tauri 会生成未签名 `.app`
- **跳过 Windows 签名**：不填 `AZURE_*` 相关 Secrets，并确保 `tauri.conf.json` 中的 `bundle.windows.signCommand` 为空或已删除


## 四、验证签名

### macOS

```bash
# 查看签名
 codesign -dv --verbose=4 /Applications/OrangeRadio.app

# 查看公证状态
 spctl -a -t exec -vv /Applications/OrangeRadio.app
```

### Windows

```powershell
# 查看签名
Get-AuthenticodeSignature "C:\Program Files\OrangeRadio\OrangeRadio.exe"
```

## 五、常见问题

### 1. macOS 公证失败：「The bundle identifier is invalid」

确保 `tauri.conf.json` 中 `identifier` 是反向域名格式，例如 `com.orangeradio.app`。

### 2. Windows Azure 登录失败

检查 Azure Service Principal 是否对 Trusted Signing Account 有 `Trusted Signing Certificate Profile Signer` 角色。

### 3. Linux 构建失败

确保 Ubuntu runner 安装了 Tauri 依赖：

```bash
sudo apt-get install -y libwebkit2gtk-4.1-dev libappindicator3-dev librsvg2-dev patchelf
```

工作流中已包含此步骤。
