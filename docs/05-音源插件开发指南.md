# 05 - 音源插件开发指南

> 如何为 OrangeRadio 开发自定义音源 · v0.1

## 一、音源体系概述

OrangeRadio 的音源是**完全可插拔**的。所有音源——本地库、网易云、QQ 音乐、Spotify、网络电台、播客——都实现统一的 [`AudioSource`](../crates/orange-core/src/source.rs) trait。

```
播放器 ──► AudioSource trait ◄── LocalSource (内置)
                        ◄── NeteaseSource (内置)
                        ◄── 你的自定义音源 ◄── 你来写
```

## 二、AudioSource trait

```rust
#[async_trait]
pub trait AudioSource: Send + Sync {
    /// 音源唯一 ID
    fn id(&self) -> SourceId;
    /// 音源类型
    fn kind(&self) -> SourceKind;
    /// 人类可读名称
    fn name(&self) -> &str;
    /// 是否需要登录鉴权（默认 false）
    fn requires_auth(&self) -> bool { false }
    /// 是否已就绪（默认 true）
    fn is_ready(&self) -> bool { true }

    /// 搜索曲目（必须实现）
    async fn search(&self, query: &SearchQuery) -> Result<SearchResult>;
    /// 解析曲目的可播放流地址（必须实现）
    async fn resolve_stream(&self, track: &Track) -> Result<StreamLocation>;

    /// 推荐（可选）
    async fn recommendations(&self, limit: u32) -> Result<Vec<Track>> { Ok(vec![]) }
    /// 用户歌单（可选）
    async fn user_playlists(&self) -> Result<Vec<PlaylistRef>> { Ok(vec![]) }
}
```

## 三、最小示例：自定义音源

以接入一个假想的 "FooMusic" 平台为例：

```rust
use async_trait::async_trait;
use orange_core::source::*;
use orange_core::track::{Track, TrackMeta};
use orange_core::Result;

pub struct FooMusicSource {
    id: SourceId,
    api_base: String,
    token: Option<String>,
}

impl FooMusicSource {
    pub fn new() -> Self {
        Self {
            id: SourceId(uuid::Uuid::new_v4()),
            api_base: "https://api.foomusic.com".into(),
            token: None,
        }
    }
}

#[async_trait]
impl AudioSource for FooMusicSource {
    fn id(&self) -> SourceId { self.id }
    fn kind(&self) -> SourceKind { SourceKind::Plugin }
    fn name(&self) -> &str { "FooMusic" }
    fn requires_auth(&self) -> bool { true }
    fn is_ready(&self) -> bool { self.token.is_some() }

    async fn search(&self, query: &SearchQuery) -> Result<SearchResult> {
        // 1. 调用平台 API
        let resp: serde_json::Value = reqwest::get(format!(
            "{}/search?q={}&page={}", self.api_base, query.keyword, query.page
        )).await.map_err(|e| CoreError::Network(e.to_string()))?
          .json().await.map_err(|e| CoreError::Network(e.to_string()))?;

        // 2. 转换为 Track 列表
        let tracks: Vec<Track> = resp["songs"].as_array()
            .unwrap_or(&vec![])
            .iter()
            .map(|s| {
                let mut track = Track::new(
                    self.id,
                    s["id"].to_string(),
                    TrackMeta {
                        title: s["name"].as_str().unwrap_or("").into(),
                        artist: s["artist"].as_str().unwrap_or("").into(),
                        duration_secs: s["duration"].as_f64(),
                        ..Default::default()
                    },
                );
                track.format = AudioFormat::Mp3;
                track
            })
            .collect();

        Ok(SearchResult {
            total: tracks.len() as u32,
            tracks,
            has_more: false,
        })
    }

    async fn resolve_stream(&self, track: &Track) -> Result<StreamLocation> {
        // 获取播放 URL
        let url = format!("{}/stream/{}", self.api_base, track.source_track_id);
        Ok(StreamLocation::Url {
            url,
            headers: vec![("Authorization".into(), self.token.clone().unwrap_or_default())],
        })
    }
}
```

## 四、注册音源

在应用启动时注册到 `SourceRegistry`：

```rust
use orange_sources::SourceRegistry;

let mut registry = SourceRegistry::new();
registry.register(Box::new(FooMusicSource::new()));
// 播放器通过 registry.get(source_id) 调度
```

## 五、StreamLocation 类型

`resolve_stream` 返回三种之一：

```rust
pub enum StreamLocation {
    /// 本地文件路径
    File { path: String },
    /// HTTP(S) 流 URL（可带请求头，如鉴权）
    Url { url: String, headers: Vec<(String, String)> },
    /// 临时 URL（带过期，需定期刷新）
    TempUrl { url: String, expires_at: DateTime<Utc> },
}
```

对于需要鉴权/防盗链的第三方平台，通常返回 `Url` 并附带 `headers`，或返回 `TempUrl` 让播放器在过期前刷新。

## 六、音源生命周期与状态

- **`is_ready()`**：返回 `false` 时，音源不出现在可用列表中，UI 提示用户去登录/配置
- **鉴权**：实现自己的 login 方法，登录后更新 token，`is_ready()` 变 true
- **错误**：用 `CoreError::AuthFailed` / `CoreError::Network` 明确区分错误类型

## 七、开发建议

1. **先读内置音源实现**：`crates/orange-sources/src/` 下的 `local.rs`、`web_radio.rs` 是最简单的参考
2. **API 请求用 reqwest**：workspace 已统一依赖
3. **错误不要 panic**：统一返回 `Result<T>`
4. **元数据尽量完整**：填全 `TrackMeta`（BPM、调性、封面），让 AI 推荐和视觉有素材
5. **测试**：可 mock `reqwest` 或针对真实 API 写集成测试

## 八、第三方平台合规提示

接入网易云、QQ 音乐等平台时：
- 优先使用**官方开放 API**（若有）
- 非官方接口存在**合规风险**，可能随时被风控
- 不得用于绕过付费/破解
- 建议作为「音源补充」而非核心，本地库 + 网络电台 + 播客是稳定合规的基础
