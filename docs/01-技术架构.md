# 01 - OrangeRadio 技术架构

> 版本：v0.3 · 最后更新：2026-07-05 · 代码总量约 15000 行（Rust 7900 + 前端 4400 + CSS 2800）

## 一、产品定位

OrangeRadio 是一款基于 **Tauri 2 + Rust** 的**全平台沉浸式智能音乐播放器**，对标超越 [Mineradio](https://github.com/XxHuberrr/Mineradio)。核心特色：多音源聚合、节奏粒子视觉、跨源歌单收藏、全屏沉浸播放。

## 三、播放架构（v0.3 实际链路）

### 跨源播放引擎（核心设计）

```
用户双击歌曲
  → engineRef.playTrack(track, index)
  → 根据 track.source_kind 分发取流方式：
      ├─ local              → convertFileSrc(文件路径) → <audio>
      ├─ netease_cloud_music → invoke("netease_stream") → URL → <audio>
      ├─ qq_music            → invoke("qqmusic_stream")
      │                        → resolve_to_file() → orangeradio://qqstream?url=...
      │                        → 自定义URI协议处理器（带Referer+Range转发）
      └─ web_radio/podcast   → URL 直接 → <audio>
  → 竞态防护：playRequestSeq 序号 + 防抖锁
  → <audio>.play() → captureStream → AnalyserNode → spectrum → 节拍检测 → 粒子视觉
```

### QQ 音乐 CORS 解决方案

QQ 音乐 CDN 不允许跨域（无 `Access-Control-Allow-Origin`），前端 `<audio>` 直接播放会被拦截。

**方案：`orangeradio://` 自定义 URI scheme**
- `apps/desktop/src-tauri/src/lib.rs` 注册异步协议处理器
- `qqmusic_stream` 返回 `orangeradio://qqstream?url=<编码URL>`
- 协议处理器用 reqwest 拉取 QQ 音乐 CDN，透传 Referer/UA/Range 头
- 支持 Range 请求（拖动进度条），流式转发不缓存整个文件

## 六、数据库设计（SQLite）

```
~/.orangeradio/library.sqlite

tracks (id TEXT PK, path TEXT, data TEXT)          -- data = Track JSON
  ├─ 本地音乐：path = 文件路径
  └─ 跨源收藏：path = 网易云/QQ 歌曲ID（可播放）

user_playlists (id TEXT PK, name, cover, created_at, updated_at)

playlist_tracks (playlist_id, track_id, added_at)  -- 多对多关联
```

**跨源收藏机制**：网易云/QQ 歌曲加入本地歌单时，完整 Track JSON（含 source_kind、artwork、meta）存入 tracks 表。播放时 engineRef 根据 source_kind 自动取流。

## 七、关键技术决策记录

### 1. 播放方案：`<audio>` + captureStream（非 MediaElementSource）
- CORS 限制导致 createMediaElementSource 静音
- 用 captureStream + createMediaStreamSource 绕过，获取真实频谱

### 2. QQ 音乐 CORS：`orangeradio://` 自定义协议
- CDN 无 Access-Control-Allow-Origin
- 注册 Tauri 异步协议处理器，流式转发带 Referer 头

### 3. QQ 音乐扫码登录：四步换证
- ptqrshow → qrsig + 二维码
- ptqrlogin → ptsigx + uin
- check_sig → p_skey（session cookie_store 自动管理）
- oauth authorize → code → QQConnectLogin.QQLogin → musicid + musickey

### 4. 竞态防护
- playRequestSeq 序号：异步取流完成后检查，过期请求放弃
- 防抖锁：同一首歌 500ms 内不重复取流

### 5. 登录态持久化
- AuthStore：keyring 托管 Master Key + AES-256-GCM 加密
- 后台循环：网易云 6h 健康检查、QQ 12h musickey 续期
